# Part 2 – NoSQL Data Management (MongoDB)

## Overview
This part of the project demonstrates the use of MongoDB for handling semi-structured data in the FlexiMart system. A product catalog is stored in JSON format, and various MongoDB operations are performed for data analysis.

## Objectives
- Store product data using a flexible schema
- Perform CRUD operations in MongoDB
- Execute aggregation queries
- Understand the advantages of NoSQL databases

## Files Description
- `products_catalog.json` – JSON file containing product catalog data
- `mongodb_operations.js` – MongoDB shell commands for CRUD and aggregation
- `nosql_analysis.md` – Analysis and explanation of NoSQL operations
